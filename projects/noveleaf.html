Noveleaf:
    Library management system

    This project was part of a team assignment for our Software Development Principles course. During the website's design phase, our team collaboratively developed structural models and activity diagrams to ensure a cohesive design.

    For my role in the project, I was responsible for designing the website's layout and implementing key functionalities related to the "Read List" and "Want to Read" features. I did not contribute to the development of preferences, recommendations, or the New York Times Best Sellers features, as those were handled by other team members.

    My Contributions:

    • Website Design: Created the overall visual structure and user interface.
    • Website Layout: Organized the front-end layout to ensure usability and a seamless user experience.
    • Backend Logic: Developed the backend functionality for managing book objects in Java.
    • Database Management: Initialized and maintained the MySQL database, ensuring efficient data handling and storage.
    • Search Functionality:
      • Integrated an API to fetch book data, which I converted into Java objects to be rendered via Thymeleaf.
      • Managed interactions with the "Want to Read" and "Read" lists, passing book objects to the database as necessary.
    • "Read Books" Feature:
      • Retrieved book data from the database and converted it into Java objects for display.
      • Integrated functionality for users to rate books, updating the database with new ratings.
    • "Want to Read Books" Feature:
      • Implemented similar functionality as the "Read Books" feature, excluding the rating logic.
    • "Most Read Books in the Last 7 Days" Feature:
      • Queried the database for books read in the last seven days, sorted them by frequency, and displayed the top seven results.
